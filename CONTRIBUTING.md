# Contributing

Contributions are welcome! This repository is a small static web app.

## How to contribute
1. Fork the repository.
2. Create a new branch:
   - `git checkout -b feature/my-improvement`
3. Make changes (HTML/CSS/JS).
4. Keep commits focused and descriptive.
5. Open a Pull Request (PR) against the `main` branch.

## Suggested improvements
- UI/UX enhancements (accessibility, responsiveness)
- Better event simulation realism
- More robust risk scoring / filtering logic
- Documentation improvements

## Code style
- Prefer readable, vanilla JavaScript (no build step required)
- Keep functions small and naming clear

## Testing
There is no automated test suite. Manually verify by opening `index.html` in a browser and checking:
- Monitoring start/stop
- Simulated attack
- Filters
- Export logs
- Clear logs
- Risk scoring and vulnerability list updates
